🧰 MCP Server & A2A Protocol

O OmniRoute roda um servidor MCP nativo (94 ferramentas, 3 transportes) e implementa o protocolo A2A para orquestração de agentes.

🚀 Início Rápido: MCP Server

Iniciar o servidor MCP (stdio)

omniroute --mcp

Conecte seu cliente MCP (ex: Cursor, Cline, Continue) via stdio transport.

Testar ferramentas MCP

Assim que conectado, use estas ferramentas no seu cliente MCP:

# Ver health do OmniRoute
omniroute_get_health

# Listar combos disponíveis
omniroute_list_combos

# Ver providers conectados
omniroute_list_providers

# Ver quotas atuais
omniroute_get_quota

# Executar compressão de contexto
omniroute_compress_context

# Ver analytics
omniroute_get_analytics
✅ 94 Ferramentas Disponíveis — Cobrem: providers, combos, quotas, logs, analytics, config, API keys, proxy, compressão, MCP, A2A, memória, skills e muito mais.

🔧 3 Transportes MCP

O OmniRoute MCP Server suporta 3 transportes simultâneos:

Transporte Como Usar Caso de Uso
stdio omniroute --mcp Clientes locais (Cursor, Cline, Continue, VS Code)
SSE omniroute --mcp-sse Clientes remotos via HTTP, integrações web
WebSocket omniroute --mcp-ws Latência ultra-baixa, streaming bidirecional

🤖 A2A Protocol (Agent-to-Agent)

O protocolo A2A permite que agentes se comuniquem diretamente via JSON-RPC 2.0. O OmniRoute expõe:

Endpoints A2A

# Agent Card (descoberta)
GET http://localhost:20128/.well-known/agent.json

# Enviar mensagem (request/response)
POST http://localhost:20128/a2a
  method: message/send

# Stream de mensagem (long-running)
POST http://localhost:20128/a2a
  method: message/stream

# Obter status de task
POST http://localhost:20128/a2a
  method: tasks/get

# Cancelar task
POST http://localhost:20128/a2a
  method: tasks/cancel

Exemplo: message/send

curl -X POST http://localhost:20128/a2a \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": "task-001",
    "method": "message/send",
    "params": {
      "skill": "quota-management",
      "messages": [
        {"role": "user", "content": "Me dê um resumo das quotas atuais"}
      ]
    }
  }'

Exemplo: message/stream (SSE)

curl -N -X POST http://localhost:20128/a2a \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": "stream-001",
    "method": "message/stream",
    "params": {
      "skill": "code-generation",
      "messages": [
        {"role": "user", "content": "Crie uma API REST em FastAPI com autenticação JWT"}
      ]
    }
  }'
# Resposta vem como Server-Sent Events (SSE)
💡 Skills Disponíveis (6): quota-management, code-generation, debugging, documentation, analysis, planning

🔍 Descoberta de Agentes

O Agent Card em /.well-known/agent.json expõe metadados para clientes externos:

curl http://localhost:20128/.well-known/agent.json

# Resposta:
{
  "name": "OmniRoute Agent",
  "description": "Universal AI Gateway with MCP & A2A",
  "url": "http://localhost:20128",
  "skills": [
    {"name": "quota-management", "description": "..."},
    {"name": "code-generation", "description": "..."}
  ],
  "capabilities": {
    "streaming": true,
    "pushNotifications": false,
    "stateTransitionHistory": true
  }
}

📊 Monitoramento MCP & A2A no Dashboard

Endpoints → A2A

MCP Management

Process health, heartbeat freshness, uptime, circuit breaker states.

🔐 Scopes MCP (10 níveis de permissão)

Controle granular do que cada cliente MCP pode fazer:

Scope Permissão
providers.readLer providers e conexões
providers.writeConectar/desconectar providers
combos.readLer combos
combos.writeCriar/editar/deletar combos
quotas.readVer quotas e limites
keys.readListar API keys
keys.writeCriar/rotacionar/deletar API keys
logs.readLer logs de requisições
analytics.readVer analytics e custos
adminAcesso total (use com cuidado)
