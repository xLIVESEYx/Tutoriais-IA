🎯 Configuração Inicial

Após instalar o OmniRoute, você precisa conectar pelo menos um provider de IA e criar uma API key para usar com suas ferramentas.

📋 Visão Geral do Dashboard

O dashboard do OmniRoute é a interface principal para gerenciar tudo. Abra no navegador:

http://localhost:20128

Páginas principais do dashboard:

🔀 Combos

Setup one-click para Claude Code, Codex, Cursor...

1️⃣ Passo 1: Conectar um Provider

Vá em Dashboard → Providers e conecte pelo menos um provider. Existem duas formas de conexão:

Opção A: OAuth (Subscription)

Para providers como Claude Code, Codex, Copilot, Kiro, Qwen, Qoder — o OmniRoute faz o fluxo OAuth 2.0 (PKCE) automaticamente:

  1. Clique no provider desejado
  2. Clique em "Connect"
  3. Faça login no navegador
  4. Os tokens são armazenados e renovados automaticamente
✅ Vantagem: Usa sua assinatura existente (Claude Pro, Codex Pro, etc.) sem custo adicional.

Opção B: API Key

Para providers como DeepSeek, Groq, xAI, Mistral, OpenAI — você fornece uma API key:

  1. Crie uma conta no site do provider
  2. Gere uma API key no painel deles
  3. No OmniRoute, clique no provider e cole a key
  4. A key é criptografada com AES-256-GCM e armazenada localmente
💡 Dica: Você pode conectar múltiplas contas do mesmo provider! O OmniRoute faz round-robin entre elas automaticamente. Quando uma atinge o limite, passa para a próxima.

2️⃣ Passo 2: Criar Sua API Key

Agora você precisa de uma API key do próprio OmniRoute para usar nas suas ferramentas de IA:

  1. Vá em Dashboard → Endpoints
  2. Clique em "Create API Key"
  3. Dê um nome descritivo (ex: "claude-code-local")
  4. (Opcional) Restrinja a key a modelos específicos:
    openai/* — todos os modelos OpenAI
    anthropic/* — todos os Claude
    if/* — apenas modelos do Kiro/Kimi
    Ou use wildcards como */gpt-4*
  5. Copie a key gerada imediatamente — ela não é mostrada novamente

3️⃣ Passo 3: Configurar um Combo (Opcional mas Recomendado)

Um combo é uma cadeia de fallback. Vá em Dashboard → Combos:

Combo zero-config: use auto

A forma mais simples — basta definir o modelo como auto e o OmniRoute constrói um combo virtual automaticamente, escolhendo o melhor provider disponível em tempo real.

Model: auto

Variantes:

  • auto — balanceado (default, LKGP)
  • auto/coding — qualidade primeiro para código
  • auto/fast — menor latência
  • auto/cheap — mais barato por token
  • auto/offline — mais quota restante
  • auto/smart — qualidade-first + 10% exploração

Combo customizado (exemplo)

Exemplo: maximizar assinatura paga + barato + fallback grátis:

Combo: "maximize-claude"
  1. cc/claude-opus-4-7          ← assinatura Claude Pro
  2. glm/glm-4.7                 ← barato ($0.6/1M)
  3. if/kimi-k2-thinking         ← grátis (Kiro)

Custo mensal: $20 + pouco backup
Resultado: alta qualidade, quase zero interrupção

Ver guia completo de Combos →

4️⃣ Passo 4: Apontar Sua Ferramenta

Configure sua ferramenta de IA (Claude Code, Cursor, Cline, etc.) com os 3 valores:

Base URL: http://localhost:20128/v1
API Key:  [cole a key do passo 2]
Model:    if/kimi-k2-thinking  (ou qualquer provider/model prefix)
💡 Cloud Endpoint: Se preferir não rodar local, pode usar http://cloud.omniroute.online/v1 com sua API key. Mas para máxima privacidade, use localhost.

Ver guia completo por ferramenta →

🧪 Validação Rápida

Teste se tudo está funcionando com curl:

Listar modelos disponíveis

curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer SUA_API_KEY"

Fazer um chat completion de teste

curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer SUA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "if/kimi-k2-thinking",
    "messages": [{"role": "user", "content": "Oi!"}]
  }'
