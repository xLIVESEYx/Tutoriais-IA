🤖 Conectar Agentes de IA ao OmniRoute

Todas as ferramentas de IA se conectam através do mesmo endpoint: http://localhost:20128/v1. Configure uma vez, use em todas.

📋 Configuração Universal (Todas as Ferramentas)

Independente da ferramenta, você sempre precisa de 3 valores:

Base URL: http://localhost:20128/v1
API Key:  [sua key do Dashboard → Endpoints]
Model:    if/kimi-k2-thinking  (ou auto, auto/coding, etc.)
💡 Dica Pro: Use auto como modelo para deixar o OmniRoute escolher o melhor provider automaticamente baseado em qualidade, custo e quota disponível. Veja Combos para mais detalhes.

🔧 Configuração por Ferramenta

🟣 OpenCode ⭐ 106K

Terminal AI agent moderno. Melhor integração nativa com OmniRoute.

Via Variáveis de Ambiente (Recomendado)

export ANTHROPIC_BASE_URL=http://localhost:20128/v1
export OPENAI_BASE_URL=http://localhost:20128/v1
export ANTHROPIC_API_KEY=sua_key_do_omniroute
export OPENAI_API_KEY=sua_key_do_omniroute

# Inicie o OpenCode
opencode

Via Config File

# ~/.config/opencode/config.json
{
  "providers": {
    "anthropic": {
      "baseUrl": "http://localhost:20128/v1",
      "apiKey": "sua_key_do_omniroute"
    },
    "openai": {
      "baseUrl": "http://localhost:20128/v1", 
      "apiKey": "sua_key_do_omniroute"
    }
  },
  "model": "auto/coding"
}

🟢 Codex CLI ⭐ 60.8K

CLI oficial da OpenAI. Funciona perfeitamente via Responses API do OmniRoute.

Variáveis de Ambiente

export OPENAI_BASE_URL=http://localhost:20128/v1
export OPENAI_API_KEY=sua_key_do_omniroute

# Inicie o Codex
codex

Config via TOML

# ~/.codex/config.toml
[model]
provider = "openai"
base_url = "http://localhost:20128/v1"
api_key = "sua_key_do_omniroute"
model = "auto/coding"
✅ Suporte completo: O OmniRoute implementa a Responses API do OpenAI, necessária para o Codex CLI funcionar corretamente.

🟣 Claude Code ⭐ 67.3K

Terminal agent da Anthropic. Requer configuração via variáveis de ambiente.

Variáveis de Ambiente

export ANTHROPIC_BASE_URL=http://localhost:20128/v1
export ANTHROPIC_API_KEY=sua_key_do_omniroute

# Inicie o Claude Code
claude

Config no .bashrc/.zshrc (Persistente)

# Adicione ao seu ~/.bashrc ou ~/.zshrc
export ANTHROPIC_BASE_URL=http://localhost:20128/v1
export ANTHROPIC_API_KEY=sua_key_do_omniroute
# Modelo padrão opcional
export ANTHROPIC_MODEL=auto/coding
ℹ️ Nota: O OmniRoute faz tradução automática de formato OpenAI ↔ Anthropic. O Claude Code acha que está falando com a API da Anthropic, mas o OmniRoute roteia para qualquer provider.

🔵 Cursor IDE Editor

Editor VS Code forkado com IA integrada. Configure nas Settings.

Via UI (Settings → Models)

  1. Abra Settings (Ctrl/Cmd + ,)
  2. Busque por "Models" ou "OpenAI"
  3. Adicione um novo modelo customizado:
Name: OmniRoute Auto
Base URL: http://localhost:20128/v1
API Key: sua_key_do_omniroute
Model: auto/coding

Via settings.json

# No settings.json do Cursor
"cursor.chat.models": [
  {
    "name": "OmniRoute Auto",
    "provider": "openai",
    "baseURL": "http://localhost:20128/v1",
    "apiKey": "sua_key_do_omniroute",
    "model": "auto/coding"
  }
]

🟠 Cline VS Code Extension

Extensão VS Code popular para agentes de IA. Suporte nativo a OpenAI-compatible endpoints.

Configuração

  1. Abra Cline (ícone na barra lateral do VS Code)
  2. Clique no ícone de configurações (engrenagem)
  3. Em API Provider, selecione "OpenAI Compatible"
  4. Preencha:
Base URL: http://localhost:20128/v1
API Key: sua_key_do_omniroute
Model: auto/coding

🐙 GitHub Copilot VS Code / IDEs

Configure via chatLanguageModels.json do VS Code. O dashboard do OmniRoute tem gerador automático.

Método 1: Gerador do Dashboard (Mais Fácil)

  1. Vá em Dashboard → CLI Tools
  2. Selecione os modelos que quer disponibilizar no Copilot
  3. Clique em "Generate Copilot Config"
  4. Copie o JSON gerado

Método 2: Manual

# Crie/edite: ~/.vscode/chatLanguageModels.json
{
  "models": [
    {
      "id": "omniroute-auto",
      "name": "OmniRoute Auto",
      "model": "auto/coding",
      "provider": "openai",
      "baseURL": "http://localhost:20128/v1",
      "apiKey": "sua_key_do_omniroute"
    },
    {
      "id": "omniroute-claude",
      "name": "OmniRoute Claude",
      "model": "if/kimi-k2-thinking",
      "provider": "anthropic",
      "baseURL": "http://localhost:20128/v1",
      "apiKey": "sua_key_do_omniroute"
    }
  ]
}

🌐 Endpoint Cloud (Opcional)

Se você não quer rodar o OmniRoute localmente, pode usar o endpoint cloud gerenciado:

Base URL: http://cloud.omniroute.online/v1
API Key:  [sua key do dashboard local ou crie conta no cloud]

🛠️ Solução de Problemas Comuns

O OmniRoute não está rodando. Execute omniroute no terminal e verifique se aparece "Server running on http://localhost:20128". Se estiver usando Docker, verifique docker ps.

A API key não confere. Vá no Dashboard → Endpoints, verifique se a key está correta e não expirou. Lembre-se: a key do OmniRoute é diferente da key do provider (OpenAI, Anthropic, etc.).

O modelo que você especificou não existe nos providers conectados. Use auto para deixar o OmniRoute escolher, ou verifique em Dashboard → Providers quais modelos estão disponíveis. Formato: provider/model (ex: if/kimi-k2-thinking, openai/gpt-4o).

Reinicie o VS Code/Cursor após alterar settings.json. Para o Copilot, o arquivo chatLanguageModels.json deve estar em ~/.vscode/ (não no workspace). Use o gerador no Dashboard → CLI Tools.

Verifique se você está usando auto/fast para priorizar latência. No Dashboard → Health, veja se algum provider está em cooldown ou circuit breaker. Tente conectar providers mais rápidos (Groq, Cerebras, Kiro).
