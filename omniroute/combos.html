🔀 Combos e Auto-Fallback

Domine o sistema de combos do OmniRoute: 18 estratégias de roteamento, auto-combos inteligentes, quota-share e playbooks para máxima performance e custo mínimo.

🎯 O que é um Combo?

Um combo é uma cadeia de modelos que o OmniRoute percorre automaticamente quando o modelo atual falha (quota esgotada, rate limit, erro, etc.). É a chave para o zero downtime.

💡 Exemplo básico: Seu combo é [cc/claude-opus-4-7 → glm/glm-4.7 → if/kimi-k2-thinking]. Se a quota do Claude acabar, ele muda para o GLM. Se o GLM acabar o budget, vai para o Kiro (grátis). Você nunca para de codar.

No dashboard, vá em Dashboard → Combos para criar, editar e ativar combos.

⚡ Zero-config: o poderoso auto

O Auto-Combo é o recurso flagship do OmniRoute. Em vez de definir manualmente a cadeia, você simplesmente define o modelo como auto (ou uma variante) e o OmniRoute constrói um combo virtual em tempo real, pontuando todos os providers conectados por 12 fatores.

Variantes do auto

auto

Balanceado padrão (LKGP - Last Known Good Path). Mantém o último provider que funcionou bem.

auto/coding

Quality-first weights otimizados para geração de código.

auto/fast

Prioriza menor latência primeiro.

auto/cheap

Prioriza menor custo por token primeiro.

auto/offline

Prioriza o provider com mais quota/rate-limit headroom restante.

auto/smart

Quality-first + 10% de exploração para descobrir melhores models.

✅ Como ativar: No dashboard, vá em Providers → [seu provider] → Keys → Edit Key e configure os parâmetros de Quota-Share.

🎛️ As 18 Estratégias de Roteamento

Ao construir combos customizados no dashboard, você pode escolher entre 18 estratégias diferentes para cada passo do combo. Estratégias definem como escolher o próximo modelo na cadeia.

# Estratégia O que faz
1 priority Lista ordenada — esgota cada alvo antes de passar para o próximo (ideal para fallback garantido).
2 fill-first Preenche a quota de cada alvo completamente antes de passar para o próximo.
3 weighted Escolha aleatória ponderada por peso definido por você.
4 round-robin Cicla pelos alvos na ordem definida.
5 p2c Power-of-two-choices: escolhe 2 aleatoriamente e escolhe o menos carregado.
6 least-used Escolhe o alvo com menor uso atual (menor carga).
7 random Escolha aleatória uniforme (deduplicada — evita repetir o mesmo imediatamente).
8 strict-random Escolha aleatória pura (pode repetir o mesmo consecutivamente).
9 cost-optimized Minimiza custo por requisição usando precificação em tempo real do catálogo.
10 headroom Escolhe o alvo com maior quota restante (mais "espaço para respirar").
11 reset-window Prefere o alvo cuja janela de reset de quota estiver mais próxima.
12 reset-aware Ranked por tempo de reset — janelas curtas primeiro (boa para quotas horárias/diárias).
13 context-relay Transfere contexto entre alvos para conversas longas (evita perda de histórico).
14 context-optimized Escolhe o melhor ajuste para o tamanho atual do contexto (prompt size).
15 lkgp Last-Known-Good Path — grudento no último alvo que funcionou (padrão do auto).
16 auto Live scoring em 12 fatores (health, quota, cost, latency, success rate, freshness...).
17 fusion Espalha para um painel de models + um juiz sintetiza uma resposta (como ensemble voting).
18 pipeline Encadeia etapas — saída de um alimenta a entrada do próximo (para tarefas complexas).
💡 Dica: Estratégias podem ser misturadas e combinadas por passo do combo. Ex: [priority → cost-optimized → context-relay].

⚖️ Quota-Share: Compartilhamento Inteligente de Assinatura

Recurso NOVO para equipes que compartilham a mesma conta upstream (ex: um plano Codex Pro usado por 3 desenvolvedores).

Em vez de uma conta ficar sem quota por causa de um pico de uso de um membro, o Quota-Share distribui o quota baseado em pesos:

⚖️ Allocation weight

Fatia da quota de cada key — ex: alice 50 / bob 30 / ci-bot 20

📐 Dimensions

O que medir: % (tempo), requests, tokens, $ — por janela 5h, 7d ou per-model

🚦 Policy

hard (bloqueia excesso), soft (deprioriza), burst (usa ocioso de outros)

🧱 Cap

Teto absoluto por key, independentemente do modo — protege contra vazamentos.

✅ Como ativar: No dashboard, vá em Providers → [seu provider] → Keys → Edit Key e configure os parâmetros de Quota-Share.
