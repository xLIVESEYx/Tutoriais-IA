📚 Referências e Comandos

Comandos CLI, variáveis de ambiente, endpoints, links oficiais e tudo que você precisa saber para usar o OmniRoute em produção.

🖥️ Comandos CLI

Todos os comandos disponíveis do OmniRoute CLI.

Comando Descrição
omniroute Inicia o servidor (API + dashboard na mesma porta 20128)
omniroute --port 3000 Define porta canônica (API) para 3000
omniroute --mcp Inicia servidor MCP com transporte stdio
omniroute --mcp-sse Inicia servidor MCP com transporte SSE
omniroute --mcp-ws Inicia servidor MCP com transporte WebSocket
omniroute --no-open Não abre o navegador automaticamente na inicialização
omniroute --help Mostra a ajuda completa com todas as opções

⚙️ Variáveis de Ambiente

Configurações avançadas via variáveis de ambiente.

Portas e Rede

Variável Default Descrição
PORT 20128 Porta da API (endpoint /v1)
DASHBOARD_PORT 20128 Porta do dashboard (use split-port mode para separar)
HOST localhost Host para bind (use 0.0.0.0 para todas as interfaces)
PROXY_URL vazio Proxy global (ex: http://proxy:8080 ou socks5://proxy:1080)
HTTP_PROXY vazio Proxy HTTP/HTTPS (alternativa a PROXY_URL)
HTTPS_PROXY vazio Proxy HTTPS (alternativa a PROXY_URL)

Timeouts

Variável Default Descrição
REQUEST_TIMEOUT_MS 600000 Timeout base para requisições upstream, TLS, API bridge
STREAM_IDLE_TIMEOUT_MS herda REQUEST_TIMEOUT_MS Gap máximo entre chunks de streaming antes de abortar
FETCH_TIMEOUT_MS herda REQUEST_TIMEOUT_MS Timeout para chegar aos headers da resposta upstream
FETCH_HEADERS_TIMEOUT_MS herda FETCH_TIMEOUT_MS Timeout para receber headers da resposta upstream
FETCH_BODY_TIMEOUT_MS herda FETCH_TIMEOUT_MS Timeout entre body chunks (0 = desativa)
FETCH_CONNECT_TIMEOUT_MS 30000 Undici TCP connect timeout
FETCH_KEEPALIVE_TIMEOUT_MS 4000 Undici idle keep-alive socket timeout
TLS_CLIENT_TIMEOUT_MS herda FETCH_TIMEOUT_MS Timeout para requisições de fingerprint TLS via wreq-js
API_BRIDGE_PROXY_TIMEOUT_MS herda REQUEST_TIMEOUT_MS ou 600000 Timeout para /v1 proxy forwarding (API→Dashboard port)
API_BRIDGE_SERVER_REQUEST_TIMEOUT_MS max(API_BRIDGE_PROXY_TIMEOUT_MS, 300000) Incoming request timeout no servidor API bridge
API_BRIDGE_SERVER_HEADERS_TIMEOUT_MS 60000 Incoming header timeout no servidor API bridge
API_BRIDGE_SERVER_KEEPALIVE_TIMEOUT_MS 5000 Keep-alive timeout no servidor API bridge
API_BRIDGE_SERVER_SOCKET_TIMEOUT_MS 0 Socket inactivity timeout (0 = desativa)

Outros

Variável Default Descrição
DATA_DIR ~/.omniroute Diretório para banco SQLite, configs, chaves, logs
DISABLE_SQLITE_AUTO_BACKUP false Desativa backup automático do banco a cada 24h
PROVIDER_LIMITS_SYNC_INTERVAL_MINUTES 70 Intervalo (min) para atualizar cache de quotas dos providers
NEXT_PUBLIC_BASE_URL http://localhost:20128 URL base para o frontend (usado em split-port mode)

🔗 Endpoints Principais

Endpoints HTTP disponíveis no servidor OmniRoute.

API Principal (OpenAI-compatible)

GET    http://localhost:20128/v1/models          # Lista modelos disponíveis
POST   http://localhost:20128/v1/chat/completions # Chat completion
POST   http://localhost:20128/v1/completions      # Text completion (legado)
POST   http://localhost:20128/v1/embeddings       # Embeddings
POST   http://localhost:20128/v1/images/generations # Geração de imagens
POST   http://localhost:20128/v1/audio/transcriptions # Transcrição de áudio
POST   http://localhost:20128/v1/audio/speech     # Text-to-speech
POST   http://localhost:20128/v1/moderations      # Moderação de conteúdo
POST   http://localhost:20128/v1/rerank           # Reranking de documentos
POST   http://localhost:20128/v1/responses        # OpenAI Responses API (Codex)

Dashboard (Web Interface)

GET    http://localhost:20128/          # Dashboard principal
GET    http://localhost:20128/dashboard/providers   # Gerenciamento de providers
GET    http://localhost:20128/dashboard/combos      # Configurar combos
GET    http://localhost:20128/dashboard/endpoints   # Gerenciar API keys
GET    http://localhost:20128/dashboard/analytics    # Visão de custos e uso
GET    http://localhost:20128/dashboard/health      # Status do sistema
GET    http://localhost:20128/dashboard/logs        # Logs unificados
GET    http://localhost:20128/dashboard/mcp         # Gerenciamento MCP
GET    http://localhost:20128/dashboard/a2a         # Gerenciamento A2A
GET    http://localhost:20128/dashboard/translator   # Playground de formatos
GET    http://localhost:20128/dashboard/settings     # Configurações gerais

Protocolos Avançados

GET    http://localhost:20128/.well-known/agent.json  # Agent Card (descoberta A2A)
POST   http://localhost:20128/a2a                     # Endpoint A2A JSON-RPC 2.0
GET    http://localhost:20128/mcp/health              # Health check do processo MCP
GET    http://localhost:20128/mcp/tools               # Lista de 94 ferramentas MCP disponíveis
