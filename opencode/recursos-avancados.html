🔬 LSP (Language Server Protocol)

Suporte nativo a 40+ linguagens com diagnóstico em tempo real.

Linguagens Suportadas

gopls (Go), pyright (Python), TypeScript-LS, rust-analyzer (Rust), e muitos outros.

{
  "lsp": true
}

Para customizar servidores LSP:

{
  "lsp": {
    "typescript": { "disabled": true }
  }
}

Tool LSP (Experimental)

Ative com OPENCODE_EXPERIMENTAL_LSP_TOOL=true:

  • goToDefinition — Vá para definição
  • findReferences — Encontre referências
  • hover — Informações sobre símbolo
  • documentSymbol — Símbolos do documento
  • workspaceSymbol — Símbolos do workspace
  • goToImplementation — Vá para implementação
  • prepareCallHierarchy — Hierarquia de chamadas

🔗 MCP (Model Context Protocol)

Servidor Local

{
  "mcp": {
    "meu-servidor": {
      "type": "local",
      "command": ["npx", "-y", "meu-mcp-command"],
      "enabled": true,
      "environment": {
        "MINHA_VAR": "valor"
      }
    }
  }
}

Servidor Remoto

{
  "mcp": {
    "sentry": {
      "type": "remote",
      "url": "https://mcp.sentry.dev/mcp",
      "oauth": {}
    }
  }
}

Autentique com opencode mcp auth sentry.

Exemplos de MCP

  • Sentry: Consulte issues e projetos do Sentry
  • Context7: Pesquise documentações técnicas
  • Grep by Vercel: Busque código no GitHub

MCP tools são carregadas por agente, economizando contexto.

🧩 Plugins

Instalar Plugins

// Via npm no opencode.json
{ "plugin": ["opencode-helicone-session", "opencode-wakatime"] }

// Via CLI
opencode plugin opencode-helicone-session

Criar Plugin

Arquivo TypeScript em .opencode/plugins/:

import type { Plugin } from "@opencode-ai/plugin"

export const MeuPlugin: Plugin = async ({ project, $ }) => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "bash") {
        console.log("Comando:", output.args.command)
      }
    }
  }
}

📦 SDK (@opencode-ai/sdk)

Instalação

npm install @opencode-ai/sdk

Uso Básico

import { createOpencode } from "@opencode-ai/sdk"

const { client } = await createOpencode({
  hostname: "127.0.0.1",
  port: 4096
})

// Criar sessão
const session = await client.session.create({
  body: { title: "Minha Sessão" }
})

// Enviar prompt
const result = await client.session.prompt({
  path: { id: session.id },
  body: {
    parts: [{ type: "text", text: "Olá!" }]
  }
})

// Listar sessões
const sessions = await client.session.list()

APIs do SDK

API Funções
session.*CRUD de sessões, prompt, command, shell, share, init
file.*Ler arquivos, status, search (text/files/symbols)
config.*Get config, list providers
tui.*Append prompt, show toast, open dialogs
auth.*Set credentials
event.*Subscribe a eventos real-time (SSE)

🎯 Agent Skills

Comportamentos reutilizáveis via arquivos SKILL.md com YAML frontmatter.

# .opencode/skills/git-release/SKILL.md
---
name: git-release
description: Cria releases e changelogs consistentes
license: MIT
---

## O que faz
- Cria release notes a partir de PRs merged
- Sugere version bump
- Comando `gh release create` pronto para copiar
💡 Skills são carregadas sob demanda via tool skill. O agente vê a lista de skills disponíveis e carrega apenas quando necessário.
