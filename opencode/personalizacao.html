🎨 Personalização

Customize cada aspecto do OpenCode: temas, configurações, regras, comandos e atalhos.

📁 Onde Fica o Config

Ordem de precedência (último vence):

  1. Remote — .well-known/opencode (organizacional)
  2. Global — ~/.config/opencode/opencode.json
  3. Custom — $OPENCODE_CONFIG
  4. Projeto — opencode.json na raiz do projeto
  5. .opencode/ — Diretório local de configuração
  6. Inline — $OPENCODE_CONFIG_CONTENT
  7. Managed — /Library/Application Support/opencode/ (admin)

Configurações são mescladas, não substituídas.

📄 Formato opencode.json

{
  "$schema": "https://opencode.ai/config.json",
  "model": "anthropic/claude-sonnet-4-5",
  "autoupdate": true,
  "server": {
    "port": 4096
  },
  "shell": "pwsh",
  "share": "manual",
  "snapshot": true
}

Schema Completo

Use o schema para autocomplete no seu editor:

  • https://opencode.ai/config.json — Server/runtime config
  • https://opencode.ai/tui.json — TUI-specific config

Criar Tema Customizado

Crie um JSON em ~/.config/opencode/themes/ ou .opencode/themes/:

{
  "$schema": "https://opencode.ai/theme.json",
  "theme": {
    "primary": { "dark": "#f97316", "light": "#ea580c" },
    "background": { "dark": "#0d1117", "light": "#ffffff" },
    "text": { "dark": "#f0f6fc", "light": "#1f2937" }
  }
}

📝 Regras (AGENTS.md)

Regras de Projeto

Crie AGENTS.md na raiz do projeto com instruções específicas:

# Meu Projeto
Este é um monorepo SST v3 com TypeScript.

## Estrutura
- `packages/` - Workspaces (functions, core, web)
- `infra/` - Definições de infraestrutura

## Comandos
- Build: `npm run build`
- Test: `npm run test`
- Lint: `npm run lint`

## Convenções
- Use TypeScript strict mode
- Código compartilhado em `packages/core/`

Regras Globais

~/.config/opencode/AGENTS.md — aplicado em todos os projetos.

Custom Instructions

Referencie arquivos de regras existentes:

{
  "instructions": [
    "CONTRIBUTING.md",
    "docs/guidelines.md",
    ".cursor/rules/*.md"
  ]
}

⌨️ Comandos Customizados

Defina comandos reutilizáveis como templates:

{
  "command": {
    "test": {
      "template": "Execute o suite de testes completo e mostre falhas.",
      "description": "Rodar testes com cobertura",
      "agent": "build",
      "model": "anthropic/claude-haiku-4-5"
    }
  }
}

Ou use Markdown em .opencode/commands/test.md.
