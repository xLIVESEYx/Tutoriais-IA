🔐 Sistema de Permissões

Cada permissão pode ser configurada como:

  • "allow" — Executa sem aprovação
  • "ask" — Pergunta antes de executar
  • "deny" — Bloqueia a ação

Configuração Global

{
  "$schema": "https://opencode.ai/config.json",
  "permission": {
    "*": "ask",
    "bash": "allow",
    "edit": "deny"
  }
}

Regras Granulares (Object Syntax)

Controle preciso com glob patterns. A última regra correspondente vence:

{
  "permission": {
    "bash": {
      "*": "ask",
      "git *": "allow",
      "npm *": "allow",
      "rm *": "deny",
      "grep *": "allow"
    },
    "edit": {
      "*": "deny",
      "packages/web/src/content/docs/*.mdx": "allow"
    }
  }
}

Diretórios Externos

Permita acesso a arquivos fora do diretório do projeto:

{
  "permission": {
    "external_directory": {
      "~/projects/personal/**": "allow"
    },
    "edit": {
      "~/projects/personal/**": "deny"
    }
  }
}

Proteção .env (padrão)

{
  "permission": {
    "read": {
      "*": "allow",
      "*.env": "deny",
      "*.env.*": "deny",
      "*.env.example": "allow"
    }
  }
}

🤖 Auto Mode

Use --auto para aprovar automaticamente permissões que não são explicitamente negadas:

opencode --auto
opencode run --auto "Refatore este módulo"

No TUI, abra a paleta de comandos e selecione "Enable auto-approve permissions".

🛡️ Proteções de Segurança

Doom Loop

Quando a mesma tool call se repete 3 vezes com input idêntico, o OpenCode solicita aprovação. Padrão: ask.

Políticas (Experimental)

Controle quais provedores podem ser usados:

{
  "experimental": {
    "policies": [
      {
        "effect": "deny",
        "action": "provider.use",
        "resource": "openai"
      }
    ]
  }
}

🔧 Custom Tools

Crie suas próprias ferramentas usando TypeScript:

// .opencode/plugins/custom-tools.ts
import { tool } from "@opencode-ai/plugin"

export const CustomToolsPlugin = async (ctx) => {
  return {
    tool: {
      saudacao: tool({
        description: "Uma ferramenta customizada",
        args: {
          nome: tool.schema.string(),
        },
        async execute(args, context) {
          return `Olá, ${args.nome}!`
        }
      })
    }
  }
}
