Visão Geral

Pi é um minimal terminal coding harness. Adapte pi aos seus fluxos de trabalho, não o contrário, sem ter que forkar e modificar pi internals. Estenda-o com TypeScript Extensions, Skills, Prompt Templates, e Themes. Coloque suas extensões, skills, prompt templates e temas em Pi Packages e compartilhe-os com outros via npm ou git.

Pi embarca poderosos padrões, mas pula features como sub agents e plan mode. Em vez disso, você pode pedir para pi construir o que quiser ou instalar um terceiro pi package que corresponda ao seu fluxo de trabalho.

Pi roda em quatro modos: interativo, print ou JSON, RPC para integração de processo, e um SDK para embutir em suas próprias apps. Veja openclaw/openclaw para uma integração de SDK em mundo real.

Filosofia

Pi é agressivamente extensível para não ditar o seu fluxo de trabalho. Features que outras ferramentas embarcam podem ser construídas com extensions, skills, ou instaladas de terceiros pi packages. Isso mantém o core minimal enquanto te deixa moldar pi para combinar como você trabalha.

Declarações Filosóficas

No MCP

Construa tools CLI com READMEs (veja Skills), ou construa uma extensão que adicione suporte MCP. Por que?

No sub-agents

Existem muitas maneiras de fazer isso. Spawne instâncias de pi via tmux, construa suas próprias com extensions, ou instale um package que faça isso do seu jeito.

No permission popups

Execute em um container, ou construa o seu próprio fluxo de confirmação com extensions inline com o seu ambiente e requisitos de segurança.

No plan mode

Escreva planos em arquivos, construa com extensions, ou instale um package.

No built-in to-dos

Eles confundem modelos. Use um arquivo TODO.md, ou construa o seu com extensions.

No background bash

Use tmux. Full observability, interação direta.

Leia o blog post para a rationale completa.

Quick Start

Instalar

npm install -g --ignore-scripts @earendil-works/pi-coding-agent

# --ignore-scripts desabilita dependency lifecycle scripts durante install.
# Pi não requer install scripts para npm install normal.

# Installer alternativa:

curl -fsSL https://pi.dev/install.sh | sh

Autenticar

export ANTHROPIC_API_KEY=sk-ant-...
pi

# Ou use sua existing subscription:

pi /login  # Então selecione o provider

Então apenas converse com pi. Por padrão, pi dá ao modelo quatro tools: read, write, edit, and bash. O modelo usa estas para cumprir seus pedidos. Adicione capabilities via skills, prompt templates, extensions, ou pi packages.

Plataformas Suportadas

Windows | Termux (Android) | tmux | Terminal setup | Shell aliases

Ferramentas

O Pi vem com padrões poderosos, mas omite features como sub agents e plan mode. Em vez disso, você pode pedir para o pi construir o que quiser ou instalar um pi package de terceiros que corresponda ao seu fluxo de trabalho.

Ferramentas embutidas

  • read - Lê o conteúdo de arquivos
  • write - Escreve conteúdo em arquivos
  • edit - Edita o conteúdo de arquivos
  • bash - Executa comandos
  • grep - via flags
  • find - via flags
  • ls - via flags

Veja a referência da CLI para opções de allowlist/exclude de ferramentas.

Providers

O Pi mantém uma lista de modelos com capacidade de tools por provider. Os catálogos são atualizados automaticamente; execute pi update --models para forçar uma atualização imediata. Autentique via subscription (/login) ou API key, e então selecione qualquer modelo daquele provider via /model (ou Ctrl+L)

Subscriptions

  • Anthropic Claude Pro/Max
  • OpenAI ChatGPT Plus/Pro (Codex)
  • GitHub Copilot
  • xAI (Grok/X subscription)
  • Radius

API Keys

Mais de 30 providers suportados via variáveis de ambiente ou arquivo de auth. Veja docs/providers.md para instruções de setup detalhadas.

Sessions

As sessions são armazenadas como arquivos JSONL com uma estrutura em árvore. Cada entry tem um id e parentId, permitindo branching in-place sem criar novos arquivos. Veja docs/session-format.md para o formato.

Management

pi -c                  # Continue a session mais recente
pi -r                  # Navegue e selecione de sessions passadas
pi --no-session        # Modo efêmero (não salvar)
pi --name "my task"    # Define o nome de exibição da session ao iniciar
pi --session  # Usa um arquivo de session específico ou ID
pi --fork     # Faz fork de um arquivo de session ou ID específico em uma nova session

Branching

/tree - Navega pela árvore de sessions in-place. Selecione qualquer ponto anterior, continue a partir dali e alterne entre branches. Todo o histórico é preservado em um único arquivo.

/fork - Cria um novo arquivo de session a partir de uma mensagem do usuário anterior no branch ativo. Abre um seletor, copia o caminho ativo até aquele ponto e coloca o prompt selecionado no editor para modificação.

/clone - Duplica o branch ativo atual em um novo arquivo de session na posição atual. A nova session mantém o histórico completo do caminho ativo e abre com o editor vazio.

Compaction

Sessions longas podem esgotar a janela de contexto. A compaction resume mensagens mais antigas enquanto mantém as recentes.

/compact ou /compact <custom instructions>

Compaction automática habilitada por padrão. Dispara em caso de overflow de contexto ou ao se aproximar do limite. Customize via extensions.

Context Files

Pi carrega AGENTS.md (ou CLAUDE.md) na inicialização a partir de:

  • ~/.pi/agent/AGENTS.md (global)
  • Diretórios ancestrais (subindo a partir do cwd)
  • Diretório atual

Use para instruções do projeto (AGENTS.md/CLAUDE.md), convenções e comandos comuns. Todos os arquivos correspondentes são concatenados.

Desabilite o carregamento de context files com --no-context-files (ou -nc).

System Prompt

Substitua o system prompt padrão com .pi/SYSTEM.md (projeto) ou ~/.pi/agent/SYSTEM.md (global). Acrescente sem substituir via APPEND_SYSTEM.md.

Customization

Prompt Templates

Prompts reutilizáveis como arquivos Markdown. Digite /name para expandir.

# ~/.pi/agent/prompts/review.md
Revise este código em busca de bugs, problemas de segurança e performance.
Foque em: {{focus}}

Coloque em ~/.pi/agent/prompts/, .pi/prompts/, ou em um pi package para compartilhar com outros.

Skills

On-demand capability packages following the Agent Skills standard. Invoke via /skill:name or let the agent load them automatically.

# ~/.pi/agent/skills/my-skill/SKILL.md
# My Skill
Use this skill when the user asks about X.

## Steps
1. Do this
2. Then that

Place in ~/.pi/agent/skills/, ~/.agents/skills/, .pi/skills/, or .agents/skills/ (from cwd up through parent directories) or a pi package to share with others.

Extensions

TypeScript modules que extend pi with custom tools, commands, keyboard shortcuts, event handlers, and UI components.

export default function (pi: ExtensionAPI) {
  pi.registerTool({ name: "deploy", ... });
  pi.registerCommand("stats", { ... });
  pi.on("tool_call", async (event, ctx) => { ... });
}

What's possible:

  • Custom tools (or replace built-in tools entirely)
  • Sub-agents and plan mode
  • Custom compaction and summarization
  • Permission gates and path protection
  • Custom editors and UI components
  • Status lines, headers, footers
  • Git checkpointing and auto-commit
  • SSH and sandbox execution
  • MCP server integration
  • Make pi look like Claude Code
  • Games while waiting (yes, Doom runs)
  • ...anything you can dream up

Place in ~/.pi/agent/extensions/, .pi/extensions/, or a pi package to share with others.

Themes

Built-in: dark, light. Themes hot-reload: modify the active theme file and pi immediately applies changes.

Pi Packages

Bundle and share extensions, skills, prompts, and themes via npm or git. Find packages on npmjs.com or Discord.

pi install npm:@foo/pi-tools
pi install npm:@foo/pi-tools@1.2.3      # pinned version
pi install git:github.com/user/repo
pi install git:github.com/user/repo@v1  # tag or commit
pi install ssh://git@github.com/user/repo
pi remove npm:@foo/pi-tools
pi list
pi update                               # update pi only
pi update --all                         # update pi and packages
pi update --extensions                  # update packages only
pi update --models                      # refresh model catalogs only
pi update npm:@foo/pi-tools             # update one package

Security

Pi is um local coding agent. Ele runs com as permissões da conta de usuário que starts it, and treats files writable por esse usuário como dentro do mesmo local trust boundary.

Project Trust

Project trust controls whether pi loads project-local settings, resources, packages, and extensions. It is not a sandbox and it does not restrict what the model pode pedir tools to do after you start working em uma directory.

Pi considers um project para ter resources que require trust when encontra any of these a partir do working directory:

  • .pi/settings.json
  • .pi/extensions, .pi/skills, .pi/prompts, or .pi/themes
  • .pi/SYSTEM.md or .pi/APPEND_SYSTEM.md
  • project .agents/skills in the current directory or an ancestor directory

No Built-in Sandbox

Pi does not include um built-in sandbox. Built-in tools podem read files, write files, edit files, and run shell commands com as permissões do processo pi.

Running Untrusted Work

For untrusted repositories, generated code you do not intend to monitor closely, or unattended automation, execute pi em um contained environment. Use um container, VM, micro-VM, remote sandbox, or policy-controlled sandbox com apenas os arquivos e credentials required for the task.

Common patterns are documented em Containerization:

  • run o whole pi process em um container/sandbox
  • run host pi enquanto routing built-in tool execution em um Gondolin micro-VM
  • mount apenas o workspace paths o agent deve access
  • avoid mounting host ~/.pi/agent unless the container deve access host sessions, settings, and credentials
  • pass o minimum required API keys or use short-lived credentials
  • restrict network access when the task does not need it
  • review diffs e outputs antes de copying results back to trusted systems

UX & Keyboard Shortcuts

Atalhos do Editor

  • Ctrl+G - Open in external editor (externalEditor, $VISUAL, $EDITOR, Notepad em Windows, ou nano elsewhere)
  • Ctrl+V - Paste uma image ou text (Alt+V em Windows), or drag images onto terminal
  • !command - Runs e sends output to LLM, !!command runs without sending
  • @ - Fuzzy-search project files
  • Ctrl+L - Open model selector
  • Shift+Tab - Cycle thinking level
  • Ctrl+T - Collapse or expand thinking blocks
  • Ctrl+O - Collapse or expand tool output
  • Ctrl+X - Copy the last assistant message

Application

  • Escape - Cancel / abort
  • Ctrl+C - Clear editor
  • Ctrl+D - Exit (quando editor vazio)
  • Alt+Enter - Queue follow-up message
  • Alt+Up - Restore queued messages to editor
  • Ctrl+P / Shift+Ctrl+P - Cycle scoped models forward/backward
  • /hotkeys - Show all atalhos keyboard

Veja docs/keybindings.md para full list and customization.

Privacy & Telemetry

Update Checks

Pi fetches https://pi.dev/api/latest-version to check se um newer Pi version existe. Disable it with PI_SKIP_VERSION_CHECK=1. Disabling update checks apenas turns off this check.

Install/Update Telemetry

After first install or a changelog-detected update, envia um anonymous version ping to https://pi.dev/api/report-install. This setting também controls optional provider attribution headers for OpenRouter, Cloudflare, and direct NVIDIA NIM requests. Opt out by setting enableInstallTelemetry to false em settings.json, or by setting PI_TELEMETRY=0. This does not disable update checks; Pi may still contact pi.dev para a latest version unless update checks are disabled or offline mode is enabled.

Offline Mode

Use --offline ou PI_OFFLINE=1 para disable all startup network operations descritos aqui, including update checks, package update checks, and install/update telemetry.

SDK & Programmatic Usage

SDK

import { createAgentSession, ModelRuntime, SessionManager } from "@earendil-works/pi-coding-agent";

const modelRuntime = await ModelRuntime.create();
const { session } = await createAgentSession({
  sessionManager: SessionManager.inMemory(),
  modelRuntime,
});

await session.prompt("What files are in the current directory?");

RPC Mode

For non-Node.js integrations, use RPC mode over stdin/stdout:

pi --mode rpc

RPC mode uses strict LF-delimited JSONL framing. Clients deve split records em \n apenas. Do not use generic line readers como Node readline, which também split em Unicode separators dentro JSON payloads.

Veja docs/rpc.md para o protocolo.
