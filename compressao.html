🗜️ Compressão de Tokens

O OmniRoute economiza 15-95% de tokens com RTK + Caveman, evitando limites de contexto e reduzindo custos.

🎯 Por que compressão de tokens importa?

Cada modelo tem um tamanho máximo de contexto (ex: 32K, 128K, 1M tokens). Quando sua conversa ultrapassa esse limite, o modelo corta partes antigas da mensagem — podendo causar:

  • ⚠️ Corte de histórico — partes antigas da conversa são removidas abruptamente
  • ⚠️ Context shift — mudança repentina no "ponto de partida" da resposta
  • ⚠️ Chamadas truncadas — resposta interrompida antes de terminar
💡 Problema real: Modelos como Claude, GPT-4o e Gemini frequentemente processam apenas os últimos ~24K tokens, mesmo quando reportam suporte a 32K/1M. Tool outputs (git diff, logs) consomem quota rapidamente.

🔧 RTK + Caveman Stacked Compression

O OmniRoute usa duas camadas de compressão em sequência (stacked) para reduzir tokens de forma inteligente:

📊 RTK (Ranked Token Killer)

Primeira camada — remove tokens "mortos" ou redundantes:

  • Boilerplate detection: Identifica e reduz estruturas repetitivas
  • Output pruning: Remove tool outputs redundantes
  • Log compression: Comprime logs de erro/extensos
  • Git diff shrink: Reduz diffs sem perder informação essencial

🪨 Caveman Compression

Segunda camada — redução agressiva para linguagens específicas:

  • Deduplication: Remove blocos idênticos consecutivos
  • Ultra mode: Compressão máxima para código/tool outputs
  • Language packs: Regras específicas para DE, FR, JA, PT, ZH, EN
  • Chinese (文言/现代): Modo wenyan para chinês clássico e moderno
✅ Resultado: Economia média de 58% em sessões tool-heavy, até 95% em inputs altamente repetitivos.

🛡️ Inflation Guard

Novo em v3.8: Por padrão, o OmniRoute descarta o resultado comprimido e envia o original se a compressão aumentaria o tamanho.

Como funciona

# Pseudo-código da lógica
if compressed_size > original_size:
    send original_payload  # Nenhum overhead
else:
    send compressed_payload
💡 Por que importa: Muitos algoritmos de compressão funcionam bem em textos longos, mas podem aumentar arquivos pequenos ou código já otimizado. O guard garante que você nunca pague mais por tentar economizar.

📊 Savings por Tipo de Conteúdo

Tipo de Conteúdo Economia Típica Estratégia Recomendada
Git diffs extensos 60-80% Caveman ultra + dedup
Logs de erro/stack traces 40-60% RTK log filter
Estruturas JSON repetitivas 50-75% RTK + Caveman dedup
Code blocks (JS/TS/Python) 30-50% Caveman language pack
Plain text/conversation 15-30% RTK basic
Outputs já compactos 0% (inflation guard) Nenhuma compressão

⚙️ Configurações de Compressão

Configure via Dashboard → Settings → Compression ou via settings.json em ~/.omniroute/:

{
  "compression": {
    "enabled": true,
    "engine": "rtk+caveman",  // ou "rtk" ou "caveman" ou "none"
    "inflation_guard": true,
    "language": "auto",  // pt, en, de, fr, ja, zh, auto
    "dedup_enabled": true,
    "git_diff_minify": true,
    "log_minify": true
  }
}
