📦 Instalação do OmniRoute

Escolha o método de instalação que melhor se adapta ao seu ambiente. O mais rápido é via npm.

🚀 Método 1: npm (Recomendado)

A forma mais simples e rápida. Requer Node.js 18+ instalado.

Instalação global

npm install -g omniroute
omniroute

✅ Dashboard abre em http://localhost:20128
✅ API disponível em http://localhost:20128/v1

Usuários pnpm

No pnpm v11, é necessário aprovar explicitamente os scripts de build nativo do better-sqlite3 e @swc/core:

pnpm add -g omniroute@latest --allow-build=better-sqlite3 --allow-build=@swc/core
omniroute

🐳 Método 2: Docker

Para isolamento total e deploy em servidores. Imagem multi-plataforma (AMD64 + ARM64).

Docker Hub

docker run -d \
  --name omniroute \
  -p 20128:20128 \
  -v omniroute-data:/root/.omniroute \
  diegosouzapw/omniroute:latest

Docker Compose

Clone o repositório e use os profiles prontos:

git clone https://github.com/diegosouzapw/OmniRoute.git
cd OmniRoute

# Profile base (sem CLI tools)
docker compose up -d

# Profile cli (com Claude Code, Codex, OpenClaw)
docker compose --profile cli up -d
💡 Volumes importantes: O diretório ~/.omniroute contém seu banco de dados, configurações e chaves criptografadas. Monte sempre como volume para persistir entre reinícios.

🐧 Método 4: Arch Linux (AUR)

Para usuários Arch Linux, existe um pacote AUR oficial que instala o OmniRoute e fornece um serviço systemd de usuário:

yay -S omniroute-bin
systemctl --user enable --now omniroute.service

📱 Método 5: Termux (Android)

Rode o OmniRoute no seu Android via Termux — útil para uso móvel ou em dispositivos sem PC.

pkg update && pkg install nodejs
npm install -g omniroute
omniroute --no-open

Acesse pelo navegador do celular em http://localhost:20128

⚙️ Configurações de Porta

Modo padrão (API + Dashboard na mesma porta)

omniroute
# API:       http://localhost:20128/v1
# Dashboard: http://localhost:20128

Split-port mode (recomendado para reverse proxy)

PORT=20128 DASHBOARD_PORT=20129 omniroute
# API:       http://localhost:20128/v1
# Dashboard: http://localhost:20129

Outros argumentos do CLI

Comando Descrição
omniroute Inicia o servidor (porta padrão 20128, API + dashboard)
omniroute --port 3000 Define a porta canônica/API para 3000
omniroute --mcp Inicia o servidor MCP (transporte stdio)
omniroute --no-open Não abre o navegador automaticamente
omniroute --help Mostra a ajuda completa

⏱️ Timeouts e Variáveis de Ambiente

Para a maioria dos deployments, as duas variáveis abaixo são suficientes:

Variável Default Função
REQUEST_TIMEOUT_MS 600000 Timeout base para upstream, TLS e API bridge
STREAM_IDLE_TIMEOUT_MS herda REQUEST_TIMEOUT_MS Gap máximo entre chunks de streaming antes de abortar
PORT 20128 Porta da API
DASHBOARD_PORT 20128 Porta do dashboard (em split-port mode)
PROVIDER_LIMITS_SYNC_INTERVAL_MINUTES 70 Intervalo de refresh do cache de quotas
⚠️ Atenção: Se você roda o OmniRoute atrás de Nginx, Caddy ou Cloudflare, garanta que os timeouts do proxy sejam maiores que os timeouts do OmniRoute.

🧪 Validação da Instalação

Após instalar, valide que tudo está funcionando:

1. Verifique se o servidor responde

curl http://localhost:20128/v1/models

2. Teste o endpoint A2A (se habilitado)

curl http://localhost:20128/.well-known/agent.json

3. Gere relatório de sistema (para debug)

npm run system-info

Gera um arquivo system-info.txt com versão do Node, OS, CLI tools instaladas, status Docker/PM2 — útil para reportar bugs.

🔄 Desinstalação

O OmniRoute oferece dois scripts de remoção limpa:

Comando Ação
npm run uninstall Remove o app, mas mantém DB e configurações em ~/.omniroute
npm run uninstall:full Remove o app e apaga permanentemente todas as configurações, chaves e bancos de dados
⚠️ Cuidado: uninstall:full é irreversível. Faça backup do diretório ~/.omniroute antes.
